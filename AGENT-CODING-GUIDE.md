# Agent Coding Guide

## 1. Résumé du projet
Agent Marketplace est une plateforme décentralisée sur Base L2 pour connecter clients et providers d'AI agents via des missions escrowées.
Le token $AGNT (ERC-20, 100M supply) gère staking (min 1,000 AGNT), fees (10% total: 3% burn, 5% insurance, 2% treasury) et gouvernance.
Fonctionnalités clés: registre d'agents (ERC-721 compliant), escrow de missions avec state machine, matching basique (tags V1, embeddings V1.5).
V1 focus: MVP core (missions, staking, escrow) sans inter-agent hiring ou ML avancé; compliance OFAC/KYC intégrée.
Objectif: Lancement mainnet post-audit, avec narrative governance-first (non deflationary).

## 4. Interfaces Solidity clés
Signatures seulement (Solidity 0.8.28, OZ v5 imports omis pour concision).

```solidity
interface IAGNTToken is IERC20, IAccessControl {
    function stake(uint256 amount) external;
    function unstake() external;
    function slash(address account, uint256 amount) external;
    function burn(uint256 amount) external;
    function stakedBalanceOf(address account) external view returns (uint256);
}

interface IAgentRegistry is IERC721, IAccessControl {
    function registerAgent(string calldata agentURI) external returns (uint256 agentId);
    function updateAgent(uint256 agentId, string calldata newURI) external;
    function getAgent(uint256 agentId) external view returns (address owner, string memory uri);
}

interface IMissionEscrow is IAccessControl {
    enum MissionState { CREATED, FUNDED, ACCEPTED, IN_PROGRESS, DELIVERED, COMPLETED, DISPUTED, REFUNDED, CANCELLED }
    function commitMission(bytes32 commitHash) external;
    function revealMission(uint256 missionId, string calldata details, uint256 value) external payable;
    function fundMission // ⚠️ USDC only (ERC-20 approve + transferFrom) — NOT payable ETH(uint256 missionId) external payable;
    function acceptMission(uint256 missionId) external;
    function startMission(uint256 missionId) external;
    function deliverMission(uint256 missionId, string calldata deliveryURI) external;
    function completeMission(uint256 missionId) external;
    function disputeMission(uint256 missionId, string calldata evidenceURI) external;
    function resolveDispute(uint256 missionId, bool clientWins) external;
    function refundMission // ⚠️ USDC only (ERC-20 approve + transferFrom) — NOT payable ETH(uint256 missionId) external;
    function cancelMission(uint256 missionId) external;
    function getMissionState(uint256 missionId) external view returns (MissionState);
    function getMissionDetails(uint256 missionId) external view returns (address client, address provider, uint256 value, MissionState state);
}

interface IInsurancePool is IAccessControl {
    function depositToPool(uint256 amount) external;
    function claimPayout(uint256 missionId, uint256 amount) external;
    function getPoolBalance() external view returns (uint256);
}
```

## 5. Schéma DB
Tables PostgreSQL avec colonnes essentielles (Prisma schema implied; enums: mission_state_enum, ofac_result_enum).

- **providers**: wallet_address (PK, text), profile (text), stake_amount (decimal), reputation_score (int default 0), status (text)
- **agents**: id (serial PK), provider_id (text FK providers), name (text), tags (text[]), metadata_cid (text), on_chain_id (text), status (text), created_at (timestamp)
- **missions**: id (serial PK), client_wallet (text), agent_id (int FK agents), state (mission_state_enum), amount_usdc (decimal), fiat_holdback_expires_at (timestamp), dispute_window_expires_at (timestamp), tx_hash (text), created_at (timestamp)
- **mission_events**: id (serial PK), mission_id (int FK missions), event_type (text), block_number (bigint), tx_hash (text), log_index (int), data (jsonb), created_at (timestamp); unique (tx_hash, log_index)
- **reviews**: id (serial PK), mission_id (int FK missions), reviewer_wallet (text), scores (jsonb), comment (text), created_at (timestamp)
- Indexes: Per table on FKs, state, tx_hash, created_at (see DB SCHEMA source for full).

## 6. Patterns obligatoires
- **Naming**: CamelCase pour Solidity vars/fns; snake_case pour DB columns; kebab-case pour API routes (e.g., /api/v1/missions/{id}/accept); PascalCase pour TS classes/components.
- **API format**: REST, /api/v1/ prefix; JSON Schema + Zod validation; Errors: { error: string, code: string, details?: object }; Rate limit: 100/min auth, 10/min anon; Webhooks on state changes.
- **State machine rules**: On-chain enum source of truth, DB mirror; Transitions: CREATED → ACCEPTED → IN_PROGRESS → DELIVERED → COMPLETED; Branches: → DISPUTED → RESOLVED; → CANCELLED (pre-ACCEPTED); → REFUNDED (post-ACCEPTED if fail); Enforce via contract requires; Sync via indexer events.

## 8. Règles NON-NÉGOCIABLES
- **Sécurité**: UUPS proxy for upgrades; Ownable2Step; ReentrancyGuard on transfer fns; SIWE + JWT auth; AES-256 for sensitive DB fields; No TEE in V1.
- **Tests**: 90% contract coverage (Hardhat Chai); API e2e with Vitest; CI must pass before merge.
- **MEV**: Commit-reveal for mission creation (hash first, reveal details); No front-running allowances.
- **OFAC**: TRM Labs screening pre-transaction; Cache Redis 1h for CLEAN; Block positives; Fail-open if TRM down with alerts; KYC at $1K/tx or $3K lifetime.
- Autres: Gas < $0.01/tx; Finality wait 2 blocks; 100% state sync DB-blockchain; No GraphQL; GDPR delete endpoint.

## 9. Démarrer le projet
```bash
# Clone repo (assume git clone <repo>)
cd agent-marketplace

# Install monorepo
yarn install

# Setup contracts
cd packages/contracts
npx hardhat init --yes  # If not bootstrapped
yarn add @openzeppelin/contracts hardhat-deploy viem
npx hardhat compile

# Setup API
cd ../api
npm init fastify -- --typescript  # If not bootstrapped
yarn add prisma zod fastify-rate-limit ioredis @trmlabs/sdk  # Add deps
npx prisma init
# Edit prisma/schema.prisma with tables from §5
npx prisma migrate dev --name init
yarn prisma generate
node server.js  # Dev server

# Setup frontend
cd ../frontend
npm create vite@latest . -- --template react-ts  # If not
yarn add wagmi viem tailwindcss postcss autoprefixer
yarn tailwindcss init
yarn dev

# Setup indexer
cd ../indexer
mkdir src && touch src/index.ts  # Bootstrap if needed
yarn add viem prisma pg
node src/index.ts  # Run

# Local chain (from root)
npx hardhat node  # Start anvil
npx hardhat deploy --network localhost  # Deploy contracts

# DB local: Assume Docker Postgres running on 5432; update .env
# Redis local: docker run -p 6379:6379 redis
```
